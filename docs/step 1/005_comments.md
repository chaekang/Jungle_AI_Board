# 005_comments

## 목적

이 문서는 [implementation_order.md](../implementation_order.md)의
`6. 댓글 기능`을 구현하기 위한 작업 가이드다.

댓글은 후기 상세 페이지를 살아 있게 만드는 기능이다.
후기 본문만으로 부족한 정보를 보충하고, 질문과 답변이 쌓이는 공간이 된다.

관련 개념 문서:

- [docs/concept/005_comments.md](../concept/005_comments.md)

## 이 단계가 끝나면 되는 것

- 댓글 작성이 된다
- 후기별 댓글 조회가 된다
- 댓글 수정 / 삭제가 된다
- 댓글 작성자 권한이 동작한다

## 범위 제한

이번 단계에서는 아래를 구현한다.

- 일반 댓글
- 후기 1개에 대한 댓글 목록
- 작성자 본인 수정 / 삭제

이번 단계에서 구현하지 않는 것:

- 대댓글
- 실시간 댓글 갱신
- 멘션
- 좋아요

## 권장 API 범위

- `POST /seat-reviews/:reviewId/comments`
- `GET /seat-reviews/:reviewId/comments`
- `PATCH /comments/:id`
- `DELETE /comments/:id`

## 권장 구현 순서

1. `comments` 테이블 또는 모델 점검
2. 댓글 DTO 정의
3. 댓글 생성
4. 후기별 댓글 목록 조회
5. 댓글 수정
6. 댓글 삭제
7. 댓글 작성자 권한 체크

## 구현 포인트

### 1. 댓글도 후기와 같은 패턴으로 간다

- 인증: Guard
- 소유권: 서비스 내부 비교
- 응답용 변환: 별도 함수

후기 CRUD와 패턴을 맞추면 유지보수가 훨씬 쉬워진다.

### 2. 정렬 기준을 미리 정한다

권장 기본값:

- 오래된 댓글부터

이유:

- 대화 흐름을 읽기 쉽다
- 초기 구현과 테스트가 단순하다

이번 단계에서 `latest` 정렬 옵션도 같이 구현한다.
후기 상세 화면에서 최신순과 오래된순을 바꿀 수 있게 맞춘다.

### 3. 후기 삭제와 댓글 처리 규칙을 정한다

후기 단계에서 `deletedAt` 기반 소프트 삭제를 쓰므로,
댓글도 같은 전략으로 맞춘다.
즉 댓글 삭제도 소프트 삭제 기준으로 구현하고,
후기가 삭제된 경우 연결 댓글도 일반 조회에서 보이지 않도록 규칙을 맞춘다.

## 완료 기준

- 후기 상세에서 댓글 흐름이 완성된다
- 댓글 권한 오류가 후기 권한 오류와 같은 방식으로 동작한다
- 후기 상세에서 필요한 기본 댓글 흐름이 안정적으로 동작한다
- 삭제 전략이 후기 단계의 삭제 방식과 일관된다
- 오래된순 / 최신순 정렬이 모두 동작한다
