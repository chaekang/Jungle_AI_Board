# 008_core_feature_integration_check

## 목적

이 문서는 [implementation_order.md](../implementation_order.md)의
`9. 기본 기능 통합 점검`을 실제 점검 항목 중심으로 정리한 문서다.

AI 기능으로 넘어가기 전에,
기본 게시판 기능이 서로 충돌 없이 붙어 있는지 확인하는 단계다.

관련 개념 문서:

- [docs/concept/008_core_feature_integration.md](../concept/008_core_feature_integration.md)

## 이 단계가 끝나면 되는 것

- 인증, 후기, 댓글, 태그, 검색 기능이 한 흐름으로 이어진다
- 응답 형식이 크게 흔들리지 않는다
- 권한 오류와 예외 처리가 일관된다
- 다음 단계인 RAG 준비가 가능하다

## 점검 대상

- 회원가입 / 로그인
- JWT 인증
- 후기 CRUD
- 댓글 CRUD
- 태그 연결
- 검색 / 필터 / 정렬 / 페이징
- 권한 체크
- 예외 처리

## 권장 점검 순서

1. 인증만 단독 점검
2. 인증 후 후기 작성 점검
3. 후기 수정 / 삭제 권한 점검
4. 댓글 작성 / 수정 / 삭제 권한 점검
5. 태그 연결 포함 후기 조회 점검
6. 검색 / 필터 / 정렬 조합 점검
7. 에러 응답 형식 점검

## 꼭 확인할 것

### 응답 형식 일관성

가능하면 아래를 흔들리지 않게 맞춘다.

- 성공 응답 구조
- 목록 응답 구조
- 에러 응답 구조

### 권한 에러 일관성

- 인증 없음: `401`
- 소유권 없음: `403`
- 대상 없음: `404`

### 삭제 정책 일관성

- 후기 삭제와 댓글 조회 결과가 모순되지 않는지 확인

## 체크리스트

- [ ] 로그인 후 후기 작성 성공
- [ ] 비로그인 후기 작성 차단
- [ ] 작성자 본인 후기 수정 / 삭제 성공
- [ ] 타인 후기 수정 / 삭제 차단
- [ ] 댓글 권한 체크 성공
- [ ] 태그 포함 상세 조회 성공
- [ ] 검색 / 필터 / 정렬 조합 응답 확인
- [ ] 공통 에러 포맷 확인

## 완료 기준

- 기본 기능을 이어서 시연해도 막히는 흐름이 거의 없다
- 이후 AI 단계에서 문제 원인을 기능 기초부와 분리해서 볼 수 있다
