# AI Board Project Plan

## 1. 프로젝트 개요

이 프로젝트는 뮤지컬 관람객이 좌석 경험을 중심으로 정보를 공유할 수 있는 게시판 서비스다.
뮤지컬은 같은 공연이라도 좌석 위치에 따라 시야, 음향, 배우 표정, 무대 장치 체감이 크게 달라지기 때문에,
관객 입장에서 가장 실질적인 정보인 "좌석 후기"를 구조적으로 모으고 검색할 수 있게 만드는 것이 핵심 목표다.

단순 후기 게시판을 넘어서, 누적된 좌석 후기를 기반으로 RAG 질의응답을 제공하고,
외부 극장 정보와 좌석 배치도를 MCP로 연동하며,
사용자 취향에 맞는 좌석을 추천하는 Agent 기능까지 확장하는 것을 목표로 한다.

## 2. 프로젝트 목표

- 극장별 좌석 후기를 쉽게 작성하고 탐색할 수 있는 게시판 제공
- 층, 구역, 열, 번호 단위까지 세분화된 좌석 경험 데이터 축적
- 시야, 음향, 배우 표정, 무대 체감 같은 관람 요소를 구조적으로 비교 가능하게 설계
- 태그, 댓글, 검색 기능을 통해 후기 활용도를 높임
- RAG 기반 질의응답으로 특정 좌석에 대한 요약 답변 제공
- 질문에 작품명이 포함되는지 여부에 따라 AI 답변 범위를 다르게 제어
- 특정 배역이나 배우를 더 잘 보기 좋은 좌우 방향 추천까지 지원
- MCP 연동으로 외부 공연장 정보와 좌석 배치도를 불러와 데이터 입력 편의성 강화
- Agent 기반 좌석 추천 기능으로 사용자의 예산과 선호도에 맞는 선택 지원

## 3. 주요 사용자

- 예매 전에 좌석 정보를 꼼꼼히 확인하고 싶은 뮤지컬 관람객
- 같은 공연을 여러 번 보며 좌석별 차이를 비교하고 싶은 회전 관객
- 배우 표정, 무대 전체 시야, 음향 밸런스 등 특정 관람 포인트를 중시하는 사용자
- 좌석 후기를 남기고 다른 관객과 의견을 나누고 싶은 커뮤니티 사용자

## 4. 해결하려는 문제

- 공식 예매처 정보만으로는 실제 시야와 체감 품질을 알기 어렵다.
- 같은 좌석군이라도 층, 구역, 열, 번호에 따라 만족도가 크게 달라진다.
- 후기 정보가 SNS, 커뮤니티, 블로그 등에 흩어져 있어 찾기 불편하다.
- "이 좌석 괜찮아?" 같은 질문이 반복되지만 기존 후기를 종합해 답해주는 기능이 부족하다.
- 공연장 좌석 배치도와 실제 후기 정보를 함께 보기가 어렵다.

## 5. 서비스 컨셉

서비스는 "뮤지컬 좌석 후기 아카이브 + AI 좌석 선택 도우미"를 지향한다.

사용자는 공연장과 좌석 위치를 기준으로 후기를 남기고,
다른 사용자는 검색과 필터를 통해 원하는 좌석 경험을 빠르게 찾을 수 있다.
이후 누적된 후기를 바탕으로 AI가 좌석 질문에 답변하고,
궁극적으로는 조건 기반 좌석 추천까지 제공한다.

### 6.1 좌석 후기 게시글

- 시야 방해 여부
  - 예: 난간, 안전바, 앞사람 머리, 무대 장치 가림
- 음향 평가
  - 예: 매우 좋음, 좋음, 보통, 아쉬움
- 좌석 편의성 평가
  - 예: 매우 편함, 편함, 보통, 불편함

## 9. RAG 기능 기획

### 9.1 목적

사용자가 특정 좌석이나 구역에 대해 자연어로 질문하면,
기존 좌석 후기들을 검색해 핵심 내용을 요약한 답변을 제공한다.

### 9.2 예시 질의

- 블루스퀘어 2층 3열 괜찮아?
- 샤롯데 1층 B구역 5열은 표정 보기 좋아?
- 드림씨어터 2층은 음향 어때?
- 지킬앤하이드 블루스퀘어 2층 3열은 어때?
- 웃는 남자 샤롯데 1층 7열은 시야 괜찮아?
- 어쩌면 해피엔딩에서 올리버 보려면 왼블이 좋아, 오른블이 좋아?
- 블루스퀘어 3층은 좌석이 불편한 편이야?

### 9.3 질의 해석 규칙

- 사용자가 좌석이나 극장 정보만 질문하면
  - 해당 좌석과 유사 좌석에 대한 전체 후기 데이터를 기준으로 답변
  - 작품을 특정하지 않은 좌석 일반론으로 요약
- 사용자가 좌석과 작품명을 함께 질문하면
  - 해당 작품의 좌석 후기만 우선 검색
  - 다른 작품 후기는 답변 근거에서 제외하거나 보조 참고 수준으로만 사용
- 사용자가 작품명과 특정 배역 또는 배우를 함께 질문하면
  - 해당 작품 후기 중 배역, 배우, 동선, 무대 좌우 체감이 언급된 후기를 우선 검색
  - 왼쪽, 오른쪽, 중앙 중 어디가 더 유리한지 방향 중심으로 요약
- 사용자가 좌석의 편안함이나 착석감을 질문하면
  - 좌석 편의성 평가와 장시간 관람 피로도 언급이 있는 후기를 우선 검색
  - 다리 공간, 등받이 체감, 장시간 착석 피로 같은 의견을 중심으로 요약
- 사용자가 작품명, 극장명, 좌석 정보 외에 회차나 날짜까지 함께 주면
  - 가능한 범위에서 더 좁은 조건으로 검색
  - 데이터가 부족할 경우 같은 작품의 인접 좌석 후기까지 확장 검색

### 9.4 동작 방식

1. 좌석 후기, 댓글, 태그, 좌석 메타데이터를 검색 가능한 문서로 수집
2. 후기 본문과 구조화된 좌석 정보를 함께 임베딩
3. 사용자 질문에서 작품명, 극장명, 층, 구역, 열, 배역명, 배우명, 편의성 관련 조건을 추출
4. 작품명이 있으면 해당 작품 후기만 우선 필터링
5. 배역명이나 배우명이 있으면 해당 키워드와 좌우 방향 관련 후기들을 우선 검색
6. 좌석 편안함 질문이면 편의성 평가와 피로도 관련 후기들을 우선 검색
7. 작품명이 없으면 좌석 위치 중심으로 관련 후기들을 검색
8. 검색된 후기들의 공통 의견과 차이를 요약
9. 답변에 근거가 된 후기 수와 대표 후기를 함께 제시

### 9.5 답변 예시 방향

- "블루스퀘어 2층 3열은 전체 무대를 보기엔 좋다는 후기가 많지만, 배우 표정 디테일은 아쉽다는 의견이 많습니다."
- "같은 구역에서도 중앙과 사이드 체감 차이가 크다는 후기가 있어 구역 정보까지 함께 확인하는 것이 좋습니다."
- "지킬앤하이드 기준으로 블루스퀘어 2층 3열은 무대 전체를 보기 좋다는 의견이 많았고, 배우 표정은 다소 멀게 느껴졌다는 후기가 많습니다."
- "작품을 특정하지 않은 질문이라 블루스퀘어 2층 3열에 대한 전체 후기 기준으로 요약했습니다."
- "어쩌면 해피엔딩에서 올리버를 더 집중해서 보고 싶다면, 후기 기준으로는 무대 왼쪽 계열 좌석이 더 낫다는 의견이 많습니다."
- "같은 작품이라도 배역 동선에 따라 왼쪽과 오른쪽 체감이 달라질 수 있어 배역 조건이 포함된 후기만 우선 반영했습니다."
- "샤롯데 2층 앞열은 시야는 괜찮지만 장시간 관람 시 다리 공간이 다소 답답하다는 후기가 있어 편한 좌석을 중시하면 신중히 보는 것이 좋습니다."

### 9.6 RAG 우선 수집 데이터

- 좌석 후기 본문
- 시야 방해 여부
- 음향 평가
- 좌석 편의성 평가
- 태그
- 댓글 중 보충 설명이 많은 내용
- 좌석 위치 메타데이터
- 작품명 및 공연 식별 메타데이터
- 배역명, 배우명, 무대 좌우 체감 관련 메타데이터

## 10. MCP 기능 기획

### 10.1 목적

외부 극장 정보, 공연장 좌석 배치도, 공연장 메타데이터를 연동해
좌석 후기 입력과 조회를 더 정확하고 편리하게 만든다.

### 10.2 활용 시나리오

- 극장 선택 시 외부 데이터에서 기본 좌석 구역 정보 불러오기
- 공연장 좌석 배치도를 조회해 후기 작성 시 좌석 위치 입력 보조
- 좌석 배치도와 후기를 함께 보여줘 사용자가 위치를 직관적으로 이해하도록 지원

### 10.3 연동 후보 데이터

- 극장 기본 정보
- 층별 좌석 구조
- 구역명 및 블록명
- 좌석 배치도 이미지 또는 구조 데이터
- 공연장별 특이사항

## 11. Agent 기능 기획

### 11.1 Agent 이름

- 좌석 선택 도우미 Agent

### 11.2 목적

사용자가 원하는 조건을 입력하면,
누적된 좌석 후기와 좌석 메타데이터를 바탕으로 적합한 좌석을 추천한다.

### 11.3 입력 예시

- 예산
- 선호 극장 또는 공연
- 보고 싶은 배우
- 더 잘 보고 싶은 배역 또는 캐릭터
- 무대 전체 위주인지 배우 표정 위주인지
- 음향을 중시하는지 여부
- 시야 방해를 얼마나 민감하게 보는지
- 좌석 편안함을 얼마나 중시하는지

### 11.4 출력 예시

- 추천 좌석 구역
- 추천 방향
  - 예: 왼쪽 블록, 오른쪽 블록, 중앙 블록
- 추천 이유
- 주의할 점
- 비슷한 대안 좌석
- 참고할 후기 링크

### 11.5 예시 시나리오

- "예산은 중간 정도고 배우 표정을 잘 보고 싶어"
- "첫 관람이라 전체 무대가 잘 보이는 좌석이 좋아"
- "최애 배우 중심으로 보고 싶은데 너무 사이드는 싫어"
- "어쩌면 해피엔딩에서 올리버를 더 잘 보고 싶은데 왼쪽이 나은지 오른쪽이 나은지 추천해줘"
- "공연이 길어서 좌석이 좀 편한 쪽으로 추천해줘"

## 12. 화면 구성 초안

## 14. 구현 우선순위 제안

4. 태그 기능

9. 좌석 선택 도우미 Agent 구현

## 15. 기대 효과

- 관람 전에 좌석 선택 실패 확률을 줄일 수 있다.
- 후기 정보가 구조화되어 극장별, 좌석별 비교가 쉬워진다.
- 반복되는 좌석 질문에 대해 AI가 빠르게 요약 답변할 수 있다.
- 외부 좌석 배치도와 후기 데이터가 결합되어 정보 신뢰도가 높아진다.
- 장기적으로는 좌석 추천 서비스로 확장할 수 있다.

## 16. 한 줄 정리

이 프로젝트는 뮤지컬 관람객의 실제 좌석 경험을 모아
검색, 후기, RAG, MCP, Agent 기능으로 확장하는
"뮤지컬 좌석 후기 중심 AI 게시판 서비스"다.
