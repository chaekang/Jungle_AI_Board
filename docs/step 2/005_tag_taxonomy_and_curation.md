# 005_tag_taxonomy_and_curation

## 현재 파일 경로 규칙

이 문서에서 코드를 추가하거나 예시 경로를 적을 때는 아래 규칙을 따른다.

- Nest 태그 도메인 코드는 `apps/nest-api/src/tags` 같은 도메인 폴더에 둔다.
- React 태그 선택/관리 UI는 `apps/web-react/src/features/tags`에 둔다.
- React 공통 HTTP 요청 함수는 `apps/web-react/src/shared/api.ts`에 둔다.
- React 화면, 컴포넌트, 스타일, 타입, 요청 코드는 기능 폴더 안에서 역할별로 나눈다.
- DTO는 Nest 도메인 폴더 아래 `dto`, 타입/인터페이스는 해당 기능 폴더의 `types.ts` 또는 `interfaces`에 둔다.

## 목적

이 문서는 2차 구현에서 태그를
"자유 입력 라벨" 수준에서 "관리 가능한 분류 체계"로 올리는 작업을 다룬다.

## 이 단계가 끝나면 되는 것

- 유사 태그와 중복 태그를 정리할 수 있다
- 추천 태그와 관리 태그를 분리할 수 있다
- 검색과 추천에서 태그 품질이 좋아진다

## 이번 단계에서 구현하는 것

- 태그 정규화 규칙
- 동의어 / 대표 태그 매핑
- 추천 태그 목록
- 관리자 태그 정리 기능

## 권장 구현 순서

1. 현재 태그 데이터 중복 패턴 분석
2. 정규화 키와 표시 이름 분리
3. 동의어 매핑 테이블 또는 대표 태그 정책 추가
4. 추천 태그 목록 정의
5. 태그 정리용 관리자 화면 또는 스크립트 추가

## 중요한 포인트

### 1. 자유 태그와 관리 태그를 분리한다

모든 태그를 완전히 통제하면 입력 장벽이 높아진다.
반대로 완전히 자유롭게 두면 검색 품질이 떨어진다.

그래서 2차에서는 보통 아래를 같이 둔다.

- 추천 태그
- 자유 입력 태그
- 관리자 정리 규칙

### 2. 태그 품질은 검색 품질과 직결된다

특히 아래 같은 태그는 정규화 이점이 크다.

- 시야 방해
- 음향 좋음
- 1열 주의
- 사이드 추천
