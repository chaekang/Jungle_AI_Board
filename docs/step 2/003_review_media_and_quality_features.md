# 003_review_media_and_quality_features

## 목적

이 문서는 2차 구현에서 후기 자체의 정보 밀도와 신뢰도를 높이기 위한
미디어, 검증, 품질 보조 기능을 다룬다.

관련 1차 문서:

- [docs/step 1/004_seat_review_crud.md](../step%201/004_seat_review_crud.md)
- [docs/step 1/003_theater_performance_seat_metadata.md](../step%201/003_theater_performance_seat_metadata.md)

## 이 단계가 끝나면 되는 것

- 후기에 이미지나 추가 증빙 정보를 붙일 수 있다
- 후기 신뢰도 판단에 도움이 되는 보조 지표가 생긴다
- 좌석 리뷰의 품질 편차를 줄일 수 있다

## 이번 단계에서 구현하는 것

- 후기 이미지 업로드 또는 이미지 URL 연결
- 스포일러 / 민감 정보 표시 플래그
- 작성 가이드 강화
- 후기 품질 점검용 필드
- 신고 누적 또는 검수 상태 필드

## 이번 단계에서 구현하지 않는 것

- 영상 업로드
- 대용량 CDN 최적화

## 권장 구현 순서

1. 후기 미디어 저장 정책 결정
2. Prisma 스키마와 응답 타입 확장
3. 업로드 검증 규칙 추가
4. 신고 / 검수 상태 필드 추가
5. 프론트 후기 작성 UI 개선

## 중요한 포인트

### 1. 업로드 기능은 저장보다 정책이 더 중요하다

아래를 먼저 정해야 한다.

- 최대 개수
- 허용 확장자
- 최대 용량
- 삭제 정책
- 신고 시 가림 처리 여부

### 2. 품질 필드는 RAG에도 직접 도움이 된다

예를 들면 아래는 나중에 검색 가중치로도 쓸 수 있다.

- 본문 길이
- 이미지 포함 여부
- 구조화 필드 충실도
- 신고 여부

## 체크리스트

- [ ] 후기 미디어 저장 방식 확정
- [ ] 업로드 검증 규칙 구현
- [ ] 품질 / 검수 상태 필드 추가
- [ ] 후기 작성 UI 반영
- [ ] 이미지 포함 후기 응답 형식 확정

## 완료 기준

- 품질이 높은 후기와 낮은 후기를 구분할 기반이 생긴다
- 후기 정보량이 1차보다 풍부해진다
- 이후 검색 / 추천 / 운영 기능과 연결할 수 있다
