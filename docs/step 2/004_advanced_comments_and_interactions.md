# 004_advanced_comments_and_interactions

## 현재 파일 경로 규칙

이 문서에서 코드를 추가하거나 예시 경로를 적을 때는 아래 규칙을 따른다.

- Nest 댓글 도메인 코드는 `apps/nest-api/src/comments` 같은 도메인 폴더에 둔다.
- React 댓글 전용 API, 훅, 컴포넌트는 `apps/web-react/src/features/comments`에 둔다.
- React 공통 HTTP 요청 함수는 `apps/web-react/src/shared/api.ts`에 둔다.
- React 화면, 컴포넌트, 스타일, 타입, 요청 코드는 기능 폴더 안에서 역할별로 나눈다.
- DTO는 Nest 도메인 폴더 아래 `dto`, 타입/인터페이스는 해당 기능 폴더의 `types.ts` 또는 `interfaces`에 둔다.

## 목적

이 문서는 2차 구현에서 댓글을
"단순 텍스트 답글"에서 "커뮤니티 상호작용 기능"으로 올리는 작업을 다룬다.

관련 1차 문서:

- [docs/step 1/005_comments.md](../step%201/005_comments.md)

## 이 단계가 끝나면 되는 것

- 대댓글이 된다
- 멘션과 좋아요가 된다
- 댓글 알림 또는 실시간 갱신 구조를 붙일 수 있다

## 이번 단계에서 구현하는 것

- 대댓글 1단계 구조
- 멘션 파싱과 표시
- 댓글 좋아요
- 최신 댓글 반영 방식 개선

## 이번 단계에서 구현하지 않는 것

- 무한 중첩 대댓글
- 복잡한 실시간 협업 편집

## 권장 구현 순서

1. 댓글 부모-자식 구조 설계
2. 댓글 API를 트리 또는 2단 구조로 확장
3. 좋아요 테이블 또는 반응 구조 추가
4. 멘션 문자열 파싱 규칙 정의
5. 폴링 또는 소켓 기반 갱신 방식 선택

## 중요한 포인트

### 1. 대댓글은 깊이 제한이 중요하다

2차에서는 보통 1단계 대댓글만 허용하는 편이 현실적이다.
무한 깊이 구조는 UI와 쿼리가 금방 복잡해진다.

### 2. 멘션은 표시와 실제 알림을 분리한다

처음부터 알림 시스템을 크게 만들지 말고,
이번 단계에서는 아래를 분리해서 생각한다.

- 텍스트 파싱
- 사용자 링크 연결
- 알림 발송

## 완료 기준

- 댓글이 단순 메모가 아니라 대화형 구조로 바뀐다
- 상호작용 데이터가 쌓여 인기 후기와 인기 댓글을 잡을 수 있다
- 이후 알림 기능으로 확장할 수 있다
