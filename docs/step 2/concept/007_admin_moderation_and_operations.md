# 007_admin_moderation_and_operations_concepts

## 이 문서의 역할

이 문서는 `007_admin_moderation_and_operations` 단계에서 필요한
운영자 기능, 신고, 검수, 소프트 삭제 활용 개념을 정리한 문서다.

사용자 기능만 있는 서비스는 빨리 만들 수 있지만,
운영 기능이 없으면 품질을 유지할 수 없다.
2차 운영 기능은 "관리자 화면 추가"가 아니라
"서비스를 유지할 수 있는 제어 장치 추가"에 가깝다.

## 핵심 개념

### `Moderation`

정의:

사용자 생성 콘텐츠를 검토하고,
서비스 정책에 맞게 노출 / 숨김 / 삭제 / 복구를 제어하는 활동이다.

언제 필요한가:

- 신고가 들어왔을 때
- 악성 콘텐츠가 생겼을 때
- 저품질 데이터가 검색을 오염시킬 때

### `Soft Delete`

정의:

데이터를 실제로 지우지 않고,
삭제된 것처럼 보이게 처리하는 방식이다.

운영에서 왜 중요한가:

- 복구 가능
- 감사 가능
- 신고 대응 가능
- 잘못된 삭제를 되돌릴 수 있음

문제 / 주의점:

- 모든 조회에서 필터를 빼먹지 않아야 한다
- 운영자 화면과 일반 사용자 화면 규칙이 달라질 수 있다

### `Report`

정의:

사용자가 문제 있는 콘텐츠를 운영자에게 알리는 구조다.

예:

- 허위 정보
- 욕설
- 스포일러
- 저작권 문제

### `Audit Log`

정의:

누가 언제 어떤 운영 조치를 했는지 남기는 기록이다.

왜 중요한가:

- 운영 실수 추적
- 팀 협업
- 정책 집행 증빙

### `Role Based Access Control`

정의:

사용자 역할에 따라 접근 가능한 기능을 나누는 방식이다.

예:

- 일반 사용자
- 운영자
- 관리자

## 면접 포인트

"운영 기능은 메인 기능이 다 끝난 다음 붙이는 부가 기능처럼 보이지만, 실제로는 소프트 삭제, 신고, 감사 로그가 없으면 서비스 품질을 유지하기 어렵습니다. 특히 사용자 생성 콘텐츠 서비스에서는 moderation 구조가 핵심입니다."

## 한 줄 정리

운영 기능은 서비스를 더 크게 만들기 위한 기능이 아니라
서비스가 망가지지 않게 붙잡아 주는 기능이다.
