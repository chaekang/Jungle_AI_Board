# 003_review_media_and_quality_features_concepts

## 현재 파일 경로 규칙

이 문서에서 코드를 추가하거나 예시 경로를 적을 때는 아래 규칙을 따른다.

- Nest 후기 도메인 코드는 `apps/nest-api/src/seat-reviews` 같은 도메인 폴더에 둔다.
- React 후기 작성/조회 화면, 훅, 컴포넌트는 `apps/web-react/src/features/reviews`에 둔다.
- React 공통 HTTP 요청 함수는 `apps/web-react/src/shared/api.ts`에 둔다.
- React 화면, 컴포넌트, 스타일, 타입, 요청 코드는 기능 폴더 안에서 역할별로 나눈다.
- DTO는 Nest 도메인 폴더 아래 `dto`, 타입/인터페이스는 해당 기능 폴더의 `types.ts` 또는 `interfaces`에 둔다.

## 이 문서의 역할

이 문서는 `003_review_media_and_quality_features` 단계에서 필요한
후기 품질, 미디어, 검수 관련 개념을 정리한 문서다.
2차에서는 후기의 수를 늘리는 것만큼
후기의 정보 밀도와 신뢰도를 높이는 것이 중요해진다.

## 핵심 개념

### `User Generated Content`

정의:

사용자가 직접 작성하거나 업로드한 콘텐츠를 뜻한다.
후기 본문, 댓글, 이미지, 태그 등 대부분이 여기에 속한다.

왜 중요한가:

- 서비스의 핵심 자산이 된다
- 동시에 품질 편차와 운영 리스크도 함께 가져온다

문제 / 주의점:

- 허위 정보
- 저품질 글
- 저작권 문제
- 부적절한 이미지 업로드

### `Review Quality Signal`

정의:

후기의 품질이나 신뢰도를 간접적으로 추정할 수 있는 신호다.

예:

- 본문 길이
- 구조화 필드 충실도
- 이미지 포함 여부
- 신고 여부
- 수정 이력

언제 쓰는가:

- 검색 랭킹 보조
- 추천 우선순위
- 운영 검수 우선순위
- RAG 문서 선택

장점:

- 단순 최신순보다 더 나은 결과를 만들 수 있다
- 저품질 후기를 걸러내는 데 도움이 된다

문제 / 주의점:

- 품질 신호가 곧 진실은 아니다
- 점수화가 과도하면 사용자 행동을 왜곡할 수 있다

### `Media Attachment`

정의:

후기에 이미지나 파일을 붙여 텍스트 외 증빙을 제공하는 구조다.

언제 쓰는가:

- 좌석 시야 예시
- 무대 거리감
- 시야 방해 위치 설명

어떻게 적용하는가:

- 파일 업로드
- 외부 URL 연결
- 업로드 검증
- 썸네일 정책

장점:

- 후기 신뢰도 상승
- 사용자 이해도 상승
- 추천 결과 설명 강화

문제 / 주의점:

- 저장 비용
- 악성 파일 업로드 위험
- 저작권 / 초상권 이슈

### `Moderation Status`

정의:

운영자가 해당 콘텐츠를 어떤 상태로 보고 있는지 나타내는 값이다.

예:

- `active`
- `hidden`
- `reported`
- `under_review`

왜 중요한가:

- 소프트 삭제와 운영 흐름을 연결한다
- 노출과 검수 정책을 코드로 다룰 수 있다

### `Spoiler Flag`

정의:

후기나 이미지에 스포일러 가능성이 있는지 표시하는 플래그다.

언제 쓰는가:

- 공연 내용 누설이 사용자 경험에 영향을 줄 때
- 리뷰 노출 UI를 조정하고 싶을 때

## 면접 포인트

"2차에서는 리뷰 양보다 리뷰 품질 관리가 중요해집니다. 텍스트 본문만 저장하면 검색과 RAG에 바로 쓰기 어렵기 때문에 이미지, 품질 신호, 검수 상태를 함께 설계하는 편이 좋습니다."

## 한 줄 정리

후기 미디어와 품질 기능은
"리뷰를 더 많이 받기 위한 기능"이 아니라
"리뷰를 더 믿고 활용하기 위한 기능"이다.
