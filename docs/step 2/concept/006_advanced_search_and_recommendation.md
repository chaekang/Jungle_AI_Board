# 006_advanced_search_and_recommendation_concepts

## 현재 파일 경로 규칙

이 문서에서 코드를 추가하거나 예시 경로를 적을 때는 아래 규칙을 따른다.

- Nest 검색 도메인 코드는 `apps/nest-api/src/search` 같은 도메인 폴더에 둔다.
- React 검색/필터 UI는 `apps/web-react/src/features/search`에 둔다.
- React 공통 HTTP 요청 함수는 `apps/web-react/src/shared/api.ts`에 둔다.
- React 화면, 컴포넌트, 스타일, 타입, 요청 코드는 기능 폴더 안에서 역할별로 나눈다.
- DTO는 Nest 도메인 폴더 아래 `dto`, 타입/인터페이스는 해당 기능 폴더의 `types.ts` 또는 `interfaces`에 둔다.

## 이 문서의 역할

이 문서는 `006_advanced_search_and_recommendation` 단계에서 필요한
고급 검색, 가중치 기반 정렬, 추천 개념을 정리한 문서다.

1차 검색은 "조건에 맞는 후기 찾기"였다면,
2차 검색은 "조건에 맞으면서 더 좋은 결과를 먼저 보여주기"가 핵심이다.

## 핵심 개념

### `Ranking`

정의:

검색 결과 후보들을 어떤 기준으로 위아래에 배치할지 정하는 과정이다.

언제 쓰는가:

- 결과는 많은데 무엇을 먼저 보여줄지 정해야 할 때
- 단순 최신순만으로 만족스럽지 않을 때

어떻게 구현하는가:

- 최근성
- 평점
- 태그 일치도
- 캐스트 일치도
- 품질 신호

장점:

- 체감 검색 품질이 크게 오른다

문제 / 주의점:

- 기준이 불명확하면 사용자 신뢰가 떨어진다
- 과도한 점수화는 디버깅을 어렵게 만든다

### `Recommendation Score`

정의:

사용자 조건과 후기 특성을 바탕으로
"얼마나 추천할 만한가"를 점수로 계산하는 값이다.

언제 쓰는가:

- 추천순 정렬
- AI 추천 후보 선정
- 개인화 결과 조정

장점:

- 복합 조건을 하나의 결과 순서로 통합할 수 있다

문제 / 주의점:

- 점수 설계가 곧 제품 철학이 된다
- 편향이 들어가기 쉽다

### `Query Normalization`

정의:

사용자 검색어를 내부 검색 기준에 맞게 정리하는 과정이다.

예:

- `1층`, `1F`
- `사이드`, `옆`
- `배우명` 표기 흔들림

### `Explainability`

정의:

왜 이 결과가 추천되었는지 설명할 수 있는 능력이다.

왜 중요한가:

- 추천은 검색보다 불투명하게 느껴진다
- RAG와 Agent 단계로 갈수록 설명 가능성이 중요해진다

면접 포인트:

"검색과 추천은 비슷해 보여도 다릅니다. 검색은 사용자의 조건을 정확히 만족시키는 것이 우선이고, 추천은 만족도 가능성이 높은 결과를 위로 끌어올리는 과정이라 ranking 기준과 explainability가 중요합니다."

## 한 줄 정리

2차 검색 고도화의 핵심은
"찾기"를 넘어 "왜 이 결과를 먼저 보여줬는지 설명 가능한 추천"을 만드는 것이다.
