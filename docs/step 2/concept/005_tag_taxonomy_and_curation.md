# 005_tag_taxonomy_and_curation_concepts

## 이 문서의 역할

이 문서는 `005_tag_taxonomy_and_curation` 단계에서 필요한
태그 정규화, 분류 체계, 큐레이션 개념을 정리한 문서다.

1차 태그가 "빠른 분류 수단"이었다면,
2차 태그는 "검색과 추천에 재사용 가능한 분류 자산"이 된다.

## 핵심 개념

### `Taxonomy`

정의:

대상을 일정한 기준으로 분류하는 체계다.
태그 시스템에서는 어떤 태그를 허용하고,
어떤 태그를 같은 그룹으로 볼지 정하는 규칙에 가깝다.

언제 쓰는가:

- 태그가 많아져 중복과 난잡함이 생길 때
- 검색 필터 품질을 안정화하고 싶을 때

장점:

- 분류 일관성이 올라간다
- 추천 태그 구성이 쉬워진다

문제 / 주의점:

- 지나치게 엄격하면 사용자가 불편하다
- 너무 느슨하면 taxonomy 의미가 없다

### `Controlled Vocabulary`

정의:

운영자가 미리 정의한 표준 용어 집합이다.

예:

- `시야좋음`
- `사이드주의`
- `음향좋음`
- `키작은사람주의`

언제 쓰는가:

- 자주 쓰이는 핵심 태그를 통제하고 싶을 때
- 검색 결과 품질을 일정하게 유지하고 싶을 때

### `Synonym Mapping`

정의:

서로 비슷한 의미의 태그를 하나의 대표 태그로 묶는 규칙이다.

예:

- `시야방해`, `시야가림`, `난간방해`

어떻게 적용하는가:

- 동의어 테이블
- 정규화 함수
- 관리자 병합 도구

장점:

- 검색 누락이 줄어든다
- 태그 데이터가 깨끗해진다

문제 / 주의점:

- 의미가 미묘하게 다른 태그를 과하게 합치면 정보가 줄어든다

### `Curation`

정의:

사용자 입력 데이터를 운영자가 선별, 정리, 병합, 추천하는 작업이다.

왜 필요한가:

- 태그는 자유도가 높을수록 데이터가 지저분해진다
- AI와 검색은 지저분한 태그를 싫어한다

## 면접 포인트

"태그 시스템은 처음엔 자유 입력으로 빨리 시작할 수 있지만, 데이터가 쌓이면 taxonomy와 synonym mapping이 없으면 검색 정확도가 급격히 떨어집니다. 그래서 2차에서는 태그를 운영 가능한 분류 체계로 올리는 게 중요합니다."

## 한 줄 정리

2차 태그 고도화는
"태그를 많이 받는 일"보다
"태그를 재사용 가능한 지식 체계로 바꾸는 일"에 가깝다.
