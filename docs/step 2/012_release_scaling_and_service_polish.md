# 012_release_scaling_and_service_polish

## 목적

이 문서는 2차 구현 마지막 단계에서
서비스를 실제 시연과 운영에 더 가깝게 다듬는 작업을 정리한 문서다.

관련 1차 문서:

- [docs/step 1/012_final_integration_and_demo.md](../step%201/012_final_integration_and_demo.md)

## 이 단계가 끝나면 되는 것

- 2차 기능까지 포함한 전체 시연 흐름이 정리된다
- 배포와 운영 체크리스트가 보강된다
- 서비스 완성도가 눈에 띄게 올라간다

## 이번 단계에서 구현하는 것

- 성능 점검
- 배포 환경 변수와 시크릿 정리
- 운영 시나리오 점검
- 사용자 경험 마감 작업
- 문서와 데모 스크립트 업데이트

## 권장 구현 순서

1. 2차 기능 포함 전체 사용자 시나리오 점검
2. 배포 설정과 시크릿 관리 재점검
3. 검색 / RAG / Agent 응답 속도 점검
4. 관리자와 일반 사용자 흐름 최종 점검
5. 데모 스크립트와 문서 업데이트

## 중요한 포인트

### 1. 2차 구현은 기능보다 일관성이 중요하다

이 단계에서는 "더 넣기"보다 아래를 우선한다.

- 기능 간 문장 톤 일관성
- 응답 형식 일관성
- 로딩 / 실패 경험 정리
- 운영 시나리오 정리

### 2. 데모 흐름은 실제 서비스 흐름과 최대한 닮아야 한다

보여주기용 기능이 아니라,
실제 사용자가 쓸 만한 흐름을 묶어 보여주는 편이 좋다.

## 체크리스트

- [ ] 2차 전체 시나리오 점검
- [ ] 배포 설정 재점검
- [ ] 응답 속도와 실패 흐름 점검
- [ ] 관리자 / 일반 사용자 흐름 점검
- [ ] 최종 문서와 데모 스크립트 업데이트

## 완료 기준

- 1차와 2차 기능을 한 번에 시연할 수 있다
- 운영 관점에서 빠진 설정이 줄어든다
- 프로젝트 완성도가 한 단계 올라간다
